# Questioning My Assumptions

> What happened was, even after correctly writing the pseudocode for inverted index building, I slipped while implementing because of an assumption I made but did not question until now.

#### TIL (21/07/2026): Things to do when I get stuck implementing something:

1. Clearly state the problem. If needed, write it down on paper or some form of a sketching app.
2. List the assumptions I've made.
3. For every assumption, ask myself: "Does this assumption actually have to be true?"
4. Reduce the problem to the smallest possible case.
5. Only then continue implementing.

Often, the bug is not in the code I write but in an assumption I never questioned.
