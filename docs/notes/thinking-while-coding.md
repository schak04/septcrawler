# Thinking while Coding

To avoid getting stuck while implementing, I've settled on the following approach:

1. I'll first think through the problem using temporary comments or pseudocode.
2. If I stop working before solving it, I'll leave WIP comments documenting the current problem, my assumptions, thought process, and next steps. I may commit these if needed and remove them once the work is complete.
3. I'll only sketch when comments are not enough to visualise things.
4. I'll keep sketches lightweight and temporary. Their purpose is to help me think, not to become documentation.

The project's design is documented in the HLD + LLD documentation and design diagrams. Sketches are only a thinking tool and may or may not be saved under `docs/sketches/`.
