# QueryFlow-related terms 

## Search Query Normalization

**Ref:** https://zilliz.com/ai-faq/what-is-search-query-normalization

Search query normalization is the process of standardizing and transforming user search queries into a more consistent format before they are processed by a search engine. This involves breaking down the queries into their essential components and converting them into a format that can be better understood and matched against available data. Normalization helps in improving search accuracy, relevance, and performance by reducing variations that might arise from different user inputs.

## Tokenization

**Ref:** https://milvus.io/ai-quick-reference/what-is-tokenization-in-fulltext-search

Tokenization in full-text search is the process of breaking down a block of text into smaller, searchable units called tokens. These tokens are typically words, numbers, or symbols, and they form the foundation of how search engines index and retrieve information. For example, the sentence “The quick brown fox jumps” might be tokenized into ["quick", "brown", "fox", “jumps”], ignoring common words like “the” (stop words) and normalizing case. Tokenization ensures that the search engine can efficiently map terms to documents, enabling fast lookups during queries. It’s the first step in text processing pipelines and directly impacts search accuracy and performance.

## Retrieval

**Ref:** https://en.wikipedia.org/wiki/Information_retrieval

Retrieval is the process where a search engine fetches the most relevant documents from its database (index) to match a user's query. It acts as the first filter, narrowing down billions of web pages into a few hundred candidates before final ranking.

## Ranking

**Ref:** https://lifesight.io/glossary/search-engine-rankings

Search engine ranking is the position your webpage occupies in search results (like Google) for specific queries. Websites that rank higher attract more visibility, organic traffic, and credibility.

## Result Assembly

**Ref:** https://www.searchenginejournal.com/search-engines/display-search-results

Result assembly (often called SERP assembly) is the final, dynamic stage of search where the engine decides exactly which links, media, and features to display on the Search Engine Results Page (SERP). It instantly gathers and structures the highest-ranked results into a single, user-friendly interface based on query intent.
